## Diagram rendering

Command:

```bash
mkdir -p docs/diagrams/svg docs/diagrams/png
for f in docs/diagrams/*.mmd; do
  base=$(basename "$f" .mmd)
  npx -y @mermaid-js/mermaid-cli \
    -p .hermes/puppeteer-config.json \
    -i "$f" \
    -o "docs/diagrams/svg/$base.svg" \
    --backgroundColor white
  npx -y @mermaid-js/mermaid-cli \
    -p .hermes/puppeteer-config.json \
    -i "$f" \
    -o "docs/diagrams/png/$base.png" \
    --backgroundColor white
done
```

Result:

- Rendered 3 SVG files and 3 PNG fallbacks successfully.
- SVG validity check passed: all files are non-empty and contain `<svg`.
- PNG validity check passed: all files are non-empty and have PNG headers.
- Markdown link-resolution check passed.
- Visual inspection: system context, request-flow, and data-model PNGs are legible and public-safe. The data-model diagram is tall but readable.
